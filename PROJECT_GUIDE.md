# Jersey Cravings Backend — Project Guide

## 1. Project Overview
Jersey Cravings is an e-commerce backend for selling football jerseys with variant-based inventory, customer accounts, admin governance, orders, and payments.

### Key Features
- **Authentication**: Email/password + Google OAuth (customer), session + access token cookies.
- **Catalog**: Categories, products, variants, product media (Cloudinary).
- **Customer**: Profile, addresses, loyalty, referrals, reviews.
- **Commerce**: Cart, coupons, orders, payment initiation, refunds, COD collection.
- **Governance**: Admin management, audit logs, bulk actions, dashboard KPIs.
- **Workers/Cron**: Payment transition cron and media cleanup worker.
- **Production safety**: Central error handler, Prisma/Zod error mapping, structured logging.

---

## 2. Tech Stack
- **Backend framework**: Node.js + Express (TypeScript)
- **Database**: PostgreSQL
- **ORM**: Prisma
- **Auth**: `better-auth` + custom JWT cookies + RBAC middleware
- **Validation**: Zod
- **Payments**: Stripe (webhook receiver + admin refund flow)
- **Media**: Cloudinary + Multer
- **Email/Templates**: Nodemailer + EJS
- **Scheduling**: node-cron
- **Logging**: Winston

### Runtime entrypoints
- **`src/app.ts`**: Express app setup, Stripe raw webhook endpoint, `/api/v1` router mount, error middleware.

### Modules (bounded contexts)
Each module follows: **route → controller → service → Prisma**

- **`auth/`**: register/login, refresh token, me, change password, logout, OTP flows, Google OAuth handlers

- **`customer/`**: profile, address, loyalty, referral, review
- **`commerce/`**: cart, coupon, order, payment, fulfillment, gift-addon
- **`governance/`**: admin, audit-log, bulk-action, dashboard

## 4. Setup Instructions

### Clone repo
```bash
git clone <your-repo-url>
cd Jersey-Cravings-backend
```

### Install dependencies
```bash
pnpm install
```

### Setup `.env`
Create a `.env` file in the project root and set all variables listed in Section 5.

### Generate Prisma client
```bash
pnpm run generate
```

### Run migrations
```bash
pnpm run migrate
```

### Run project (development)
```bash
pnpm run dev
```

### Run project (production)
```bash
pnpm run build
```

## 5. Environment Variables
All variables below are required by `src/app/config/env.ts`.

### Core
- **`NODE_ENV`**: `development` | `production`
- **`PORT`**: API port (example: `5000`)
- **`FRONTEND_URL`**: frontend origin allowed by CORS
- **`DATABASE_URL`**: PostgreSQL connection string

### Better Auth
- **`BETTER_AUTH_SECRET`**: Better Auth secret
- **`BETTER_AUTH_URL`**: Better Auth base URL (also allowed by CORS)
- **`BETTER_AUTH_SESSION_TOKEN_EXPIRES_IN`**: session token expiry config
- **`BETTER_AUTH_SESSION_TOKEN_UPDATE_AGE`**: session refresh/update age config

### JWT cookies
- **`ACCESS_TOKEN_SECRET`**: JWT access token secret
- **`REFRESH_TOKEN_SECRET`**: JWT refresh token secret
- **`ACCESS_TOKEN_EXPIRES_IN`**: access token expiry (example: `15m`)
- **`REFRESH_TOKEN_EXPIRES_IN`**: refresh token expiry (example: `7d`)

### Email (Nodemailer)
- **`EMAIL_SENDER_SMTP_USER`**: SMTP username
- **`EMAIL_SENDER_SMTP_PASS`**: SMTP password/app password
- **`EMAIL_SENDER_SMTP_HOST`**: SMTP host
- **`EMAIL_SENDER_SMTP_PORT`**: SMTP port
- **`EMAIL_SENDER_SMTP_FROM`**: From address (example: `Jersey Cravings <no-reply@domain.com>`)

### Google OAuth
- **`GOOGLE_CLIENT_ID`**: Google OAuth client id
- **`GOOGLE_CLIENT_SECRET`**: Google OAuth secret
- **`GOOGLE_CALLBACK_URL`**: OAuth callback URL

### Cloudinary
- **`CLOUDINARY_CLOUD_NAME`**: Cloudinary cloud name
- **`CLOUDINARY_API_KEY`**: Cloudinary API key
- **`CLOUDINARY_API_SECRET`**: Cloudinary API secret

### Super Admin seed
- **`SUPER_ADMIN_EMAIL`**: seed account email
- **`SUPER_ADMIN_PASSWORD`**: seed account password

### Stripe
- **`STRIPE_SECRET_KEY`**: Stripe secret key
- **`STRIPE_WEBHOOK_SECRET`**: Stripe webhook secret for signature verification

### Worker / Cron
- **`ENABLE_MEDIA_CLEANUP_WORKER`**: set to `false` to disable the media cleanup worker
- **`MEDIA_CLEANUP_CRON_SCHEDULE`**: cron schedule (server default is `0 * * * *`)

## 6. Authentication & Authorization

### How authentication works
- Custom auth routes live under **`/api/v1/auth/*`**.
- Better Auth adapter routes are mounted under **`/api/auth/*`**.
- Protected APIs require cookies:
  - `accessToken` (JWT)
  - `better-auth.session_token` (session token validated in DB)

### RBAC and middleware
Protected routes use **`checkAuth(...roles)`** (see `src/app/middleware/checkAuth.ts`) to validate:
- session existence/expiry + user status
- access token signature/claims + role authorization

### Roles
- **`CUSTOMER`**: storefront + own resources
- **`ADMIN`**: operational/admin resources
- **`SUPER_ADMIN`**: elevated governance (admin creation, role changes)

## 8. Background Jobs / Cron

### 8.1 Media cleanup worker (Cloudinary cleanup)
- Started from `src/server.ts` via `startMediaCleanupWorker(...)`.
- **Enable/disable**: `ENABLE_MEDIA_CLEANUP_WORKER` (disabled only when set to `false`)
- **Schedule**: `MEDIA_CLEANUP_CRON_SCHEDULE` (server fallback default: `0 * * * *`)
- Responsibility: process Cloudinary cleanup queue and delete remote media after enqueueing.

### 8.2 Payment/order status transition cron
- Initialized in `src/app.ts` via `initPaymentCron()`.
- Schedule in code: **every 15 minutes** (`*/15 * * * *`)
- Responsibility:
  - auto-cancel stale unpaid Stripe orders and **release inventory**
  - mark stuck processing orders for manual review
  - archive old failed/cancelled orders

## 9. Error Handling
All errors flow through `src/app/middleware/globalErrorHandler.ts`:
- structured logging of error + request metadata
- cleanup of uploaded files on failure
- normalized response for Prisma errors, Zod errors, `AppError`, and generic errors
- stack traces are exposed only in `development`

## 10. Logging System
Winston logger lives in `src/app/lib/logger.ts`:
- file outputs: `logs/error.log`, `logs/combined.log`, `logs/exceptions.log`, `logs/rejections.log`
- console logging enabled when `NODE_ENV !== "production"`
